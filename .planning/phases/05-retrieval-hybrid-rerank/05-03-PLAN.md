<interfaces>
Input contract (verified against Phase 4 protocols at `packages/docintel-core/src/docintel_core/adapters/protocols.py`):
- `Bm25sStore.query(query_text: str, k: int) -> list[tuple[str, int, float]]`
- `NumpyDenseStore.query(q: np.ndarray, k: int) -> list[tuple[str, int, float]]`
- `QdrantDenseStore.query(q: np.ndarray, k: int) -> list[tuple[str, int, float]]`
All three return `list[(chunk_id, rank, score)]` where rank is 0-BASED.

Output contract (per D-07 + RESEARCH §1):
- `_rrf_fuse(bm25_results, dense_results, k=60) -> list[tuple[str, float]]`
- Returns `list[(chunk_id, rrf_score)]` sorted DESCENDING by rrf_score, with BM25-rank ASCENDING as tie-break (CD-05).

Formula (Cormack 2009, verbatim):
- `rrf_score(chunk_id) = sum over rankers r of [ 1 / (k + rank_r_1based(chunk_id)) ]`

Hand-computed assertion values:
- One chunk at rank 0 (1-based: 1) in BOTH rankers, k=60: `2 * (1/61) ≈ 0.0327869`.
- One chunk at rank 0 in BM25 only, k=60: `1/61 ≈ 0.0163934`.
- One chunk at rank 99 (1-based: 100) in both, k=60: `2 * (1/160) = 0.0125`.

Skip-the-contribution rule (RESEARCH §1):
- Iterate `set(bm25_ranks) | set(dense_ranks)`.
- For each chunk_id, contribute `1.0 / (k + 1based_rank)` ONLY from rankers where it is present.

Tie-break (CD-05):
- Primary sort key: `-rrf_score` (descending).
- Secondary sort key: BM25 rank ascending; `float("inf")` if BM25-missing (pushes BM25-missing chunks after BM25-present ties).
