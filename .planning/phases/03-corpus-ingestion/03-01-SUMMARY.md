## Decisions Made

- **Pre-condition assertions guard against xpass false-positives.** When `scripts/check_ingest_wraps.sh` is missing, `bash <missing>` returns 127, and a naive `assert result.returncode != 0` would xpass (the test would "pass" without exercising the gate). Adding `assert _GATE_SCRIPT.is_file()` before the subprocess invocation surfaces the missing artifact as the xfail reason. Same pattern applied to `test_corpus_cache_gitignored` (which checks for the `!data/corpus/raw/` negation marker in `.gitignore`) and `test_no_torch_import_on_help` (which asserts the CLI exit 0 + populated subcommands before inspecting PYTHONVERBOSE output). All three would have produced misleading xpass markers before this hardening; all three now xfail correctly.
- **Module-level `_XFAIL` constant.** Each test file declares `_XFAIL = pytest.mark.xfail(raises=(...), strict=False, reason="awaits Wave N — ...")` once and decorates every test with `@_XFAIL`. Eliminates repeated boilerplate and makes the wave-flip commit (Wave 1/3/4/5) trivial — a single edit removes the constant.
- **Placeholder JSON/JSONL fixtures use schema-shaped data with `_comment` markers.** Wave 3 / Wave 4 strip the `_comment` key when regenerating real bytes; the marker signals to any human reader that the file is a placeholder, not a frozen golden.
- **Sample HTML uses synthetic prose only.** No SEC content reproduced (T-3-V14-02 disposition: accept). 5.5 KB well under the 30 KB target.
